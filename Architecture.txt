BADFENG-DUMMY PROJECT ARCHITECTURE (Interview Ready Notes)
===========================================================

1) Project Overview
-------------------
Yeh project Next.js 14 (App Router) + TypeScript + Tailwind CSS pe based ek ecommerce-style homepage clone hai.
Goal: Figma/screenshot-based UI ko desktop + mobile me close visual match dena.

Main business intent:
- Walkie-talkie products showcase
- Category browsing
- Promo + blogs + trust features
- Footer with support/contact/download app info


2) Tech Stack and Why
---------------------
- Next.js 14.2.25 (App Router): file-based routing, server/client rendering support, optimized build.
- React 18: component-driven UI.
- TypeScript: better type safety and interview me architecture clarity.
- Tailwind CSS: fast utility-first styling, especially screenshot-accurate spacing/sizing ke liye useful.
- next/image: image optimization + layout control.
- next/font/google: font loading optimization (Sora, Space Grotesk).

Why good interview answer:
"Humne Tailwind use kiya kyunki design highly custom aur pixel-iterative tha; utility classes se fast iteration hua."


3) High-Level Architecture
--------------------------
Rendering flow:
- src/app/layout.tsx: global shell, fonts, metadata
- src/app/page.tsx: homepage composition (section orchestration)
- src/components/*.tsx: section-wise reusable UI blocks
- public/*: local assets (png/svg), screenshot-matching visuals

Architecture style:
- Presentational component architecture
- Static data + asset driven UI
- Minimal business logic, heavy visual composition


4) Folder Structure (Important)
--------------------------------
- src/app/
  - layout.tsx (root layout + fonts + hydration warning suppression)
  - page.tsx (home page component composition)
  - globals.css (global theme + base overflow guard)

- src/components/
  - Header.tsx
  - HeroSection.tsx
  - Categories.tsx
  - SaleTimer.tsx
  - TrendingProducts.tsx
  - ProductGallery.tsx
  - Accessories.tsx
  - PromoBanner.tsx
  - FeatureHighlights.tsx
  - Footer.tsx

- src/data/
  - products.ts
  - categories.ts

- src/types/
  - product.ts (Product, Category types)

- public/
  - section assets (HeroSections, GroupFour, SaleTrendingGroup, etc.)
  - icons: UpArrow.svg, Debit.svg, Valut.svg, return.svg, SendArrow.svg


5) Page Composition (Top to Bottom)
------------------------------------
src/app/page.tsx me order:
1. Header
2. HeroSection
3. Categories
4. SaleTimer + TrendingProducts + side image row
5. ProductGallery + Accessories row
6. PromoBanner
7. FeatureHighlights (Our Blogs + service features)
8. Footer

Interview line:
"Page orchestrator ke roop me page.tsx sirf composition handle karta hai, section logic component level pe rehta hai."


6) Component Responsibilities
-----------------------------
Header.tsx
- Desktop aur mobile ke liye alag navbar layout
- logo, cart icon, profile icon

HeroSection.tsx
- Main hero card with product image, floating icons, CTA
- Mobile me simplified focus + desktop me additional side cards

Categories.tsx
- "More Categories" fixed card + internal horizontal carousel on mobile
- Separate mobile carousel cards for Featured and Professional series
- Key challenge: clipping and overflow handling

SaleTimer.tsx
- Sale badge, title, timer blocks, product image, arrow CTA

TrendingProducts.tsx
- Trending card with badge, main visual, play button (mobile), dots indicator
- Desktop dots alignment center tuned

ProductGallery.tsx
- Walkie Talkie grid card (2-column tiles)

Accessories.tsx
- Product detail card with stars, price, buy/add buttons

PromoBanner.tsx
- Mobile: top image + bottom blue content panel
- Desktop: image background + overlay content

FeatureHighlights.tsx
- "Our Blogs" section
- Mobile me only first blog card visible
- Feature strip icons now SVG asset-based from public folder

Footer.tsx
- Mobile-first rounded dark card layout
- Newsletter input with SendArrow.svg
- Support, links, app download, social icons


7) Data and Typing Layer
------------------------
src/types/product.ts:
- Product interface
- Category interface

src/data/products.ts and src/data/categories.ts:
- Static mock data
- Local UI sections mostly local assets use karte hain; data files interview me architecture completeness dikhate hain

Interview line:
"Agar backend integrate karna ho to same interfaces API contracts ban sakte hain."


8) Styling Strategy
-------------------
- Tailwind utilities with breakpoint prefixes
- Mobile-first classes + sm/md/lg overrides
- Screenshot matching ke liye custom pixel values used (rounded, h/w, spacing)
- Reusable color family maintained around light blue-gray surfaces + dark footer

Important pattern:
- Mobile me rounded card style + border
- Desktop me kuch cards cleaner flat style


9) Responsive Design Strategy
-----------------------------
Core breakpoints used:
- default: mobile
- sm: tablet/small desktop transitions
- md/lg: multi-column desktop

Responsive decisions:
- Horizontal carousels only inside card containers
- `overflow-x-hidden` guards page level
- `min-w-0` in grid items where needed to avoid clipping
- Fixed mobile card heights where screenshot demanded stable framing

Interview point:
"Humne intentional fixed heights sirf un sections me rakhe jahan art-direction strict tha, baaki jagah fluid width/stack behavior maintain kiya."


10) Image/Icon System
---------------------
- Product visuals mostly local assets from public/
- next/image used for optimization and CLS control
- Arrow glyphs replaced globally by /UpArrow.svg
- Footer input button uses /folder/SendArrow.svg
- Feature icons use SVG assets: /return.svg, /Debit.svg, /Valut.svg, etc.


11) Key Issues Faced and Fixes (Very Interview-Relevant)
---------------------------------------------------------
Issue 1: Missing Next chunk/module error (`Cannot find module './948.js'`)
Cause: stale `.next` cache artifacts
Fix: delete .next and restart dev/build

Issue 2: Hydration warning (`Extra attributes from the server: cz-shortcut-listen`)
Cause: browser extension injecting attributes
Fix: `suppressHydrationWarning` on html/body in layout.tsx

Issue 3: Mobile card clipping in Categories
Cause: overflow and grid min-width interactions
Fix: add min-w-0, constrain internal carousel width, remove problematic outer clipping

Issue 4: Desktop promo content not visible enough
Cause: heavy overlay + layering
Fix: lighter desktop gradient + z-index for content + min-height

Issue 5: Dots alignment mismatch in Trending card desktop
Fix: indicator row separated and centered


12) Performance Notes
---------------------
- next/image used across sections
- local assets reduce remote dependency and improve consistency
- Build validated regularly with `npm run build`

Potential improvement:
- reduce overly large assets
- convert repeated PNGs to modern formats (if quality allows)


13) Accessibility Notes
-----------------------
- Buttons have visible controls and mostly aria labels on icon-only buttons
- Alt text present for major images/icons
- Could improve further with better semantic heading hierarchy and form submit behavior


14) Commands and Build/Run Flow
-------------------------------
- Dev: npm run dev
- Build: npm run build
- Start: npm run start
- Lint: npm run lint

Troubleshooting standard steps:
1. Stop dev server
2. Remove .next
3. Re-run npm run dev / npm run build


15) Interview Questions You Can Expect (and what to answer)
------------------------------------------------------------
Q1. Why Next.js App Router for this project?
A: Better file-based routing, server/client flexibility, and optimized build pipeline.

Q2. Why Tailwind instead of plain CSS?
A: Screenshot-driven rapid iteration, fast spacing and responsive adjustments.

Q3. How did you ensure mobile responsiveness?
A: Mobile-first classes, breakpoint overrides, container-bound carousels, overflow guards.

Q4. How did you handle design mismatch iterations?
A: Section-wise refinement + frequent build validation + controlled pixel tuning.

Q5. What architecture pattern did you use?
A: Presentational component architecture with page-level composition.

Q6. How to scale this project if backend comes in?
A: Replace static data files with API layer, keep types as contract, split UI into domain modules.

Q7. Biggest bug and fix?
A: `.next` stale chunk issue (`./948.js`) fixed by cache clean + restart.

Q8. How to avoid hydration mismatch warnings?
A: Check extension-injected attributes, use suppressHydrationWarning where appropriate.

Q9. Why local assets instead of remote URLs?
A: Pixel-consistent design cloning and stable rendering across environments.

Q10. If design system banana ho to next step?
A: Extract repeated values into tokens/components (Card, Badge, CTAButton, IconCircle).

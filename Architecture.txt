BADFENG-DUMMY PROJECT ARCHITECTURE (Latest Interview Notes)
===========================================================

1) Project Snapshot
-------------------
Yeh project Next.js 14 App Router based responsive ecommerce-style homepage clone hai.
Primary goal: Figma/screenshot ke close visual match ke saath smooth desktop + mobile UX deliver karna.

Core outcomes achieved:
- Section-based homepage architecture
- Asset-driven UI (local images + SVG)
- Mobile + desktop parity tuning
- GSAP-based reveal/hover micro-interactions
- Multiple auto-carousels with manual controls
- Scroll performance optimization (lag reduction)


2) Final Tech Stack
-------------------
- Next.js 14.2.25 (App Router)
- React 18
- TypeScript
- Tailwind CSS
- GSAP + ScrollTrigger
- next/image
- next/font/google (Sora + Space Grotesk)

Why this stack (interview line):
"Design highly iterative tha, isliye Tailwind + component architecture se fast visual refinement possible hua, aur Next.js ne optimized build + routing structure diya."


3) High-Level Architecture
--------------------------
Top-level flow:
- `src/app/layout.tsx`: global shell, metadata, fonts, hydration-warning suppression
- `src/app/page.tsx`: homepage composition/orchestration
- `src/components/*`: isolated section components
- `src/data/*`: static seed/mock data
- `public/*`: all icons and product/blog assets

Architecture style:
- Component-driven presentational architecture
- Thin page orchestrator, section-level logic inside components
- Asset-first rendering for screenshot fidelity

7) Animation System Design
--------------------------
Animation pattern used:
- Marker-based triggers via attributes:
  - `data-gsap="fade-up" | "fade-down"`
  - `data-gsap-card`
  - `data-gsap-hover`
  - `.gsap-hover-image`

Global animation controller:
- `GsapAnimations.tsx` applies:
  - section reveal on scroll
  - card reveal
  - hover lift + image scale (leaf cards only)

Performance-aware behavior (important):
- Mobile par heavy loops disabled/reduced:
  - hero floating icons disabled on mobile
  - CTA pulse disabled on mobile
  - promo parallax disabled on mobile
- Shorter animation durations for mobile reveal
- `ScrollTrigger.config({ ignoreMobileResize: true })`
- `gsap.defaults({ force3D: true })`

9) Responsive Strategy (Final)
------------------------------
- Mobile-first Tailwind classes
- `sm`, `md`, `lg` breakpoints for structure changes
- Controlled fixed heights only where art-direction required
- Full-page overflow protection: `overflow-x-hidden` at root
- Internal carousels constrained inside section boundaries
- Desktop 3-card row parity maintained with equalized heights

11) Major Issues Faced and Final Fixes
--------------------------------------
Issue A: `Cannot find module './948.js'`
- Root cause: stale `.next` build chunks
- Fix: clear `.next` and restart dev server
- Sometimes old Node process on port 3000 also caused stale behavior; process cleanup done

Issue B: hydration warning from extension attribute (`cz-shortcut-listen`)
- Fix: `suppressHydrationWarning` on `html` and `body` in `layout.tsx`

Issue C: hover effect parent section par trigger ho raha tha
- Fix: hover targets reduced to leaf-card level only

Issue D: mobile featured carousel static feel
- Fix: state-driven index + snap scroll + controlled autoplay

Issue E: mobile promo text low contrast
- Fix: heading/paragraph colors brightened and copy typo corrected

Issue F: scroll lag / stuck feel
- Fixes combined:
  - heavy mobile animations disabled
  - visibility-aware timers
  - autoplay pause on interaction
  - reduced GSAP workload on mobile


12) Build and Run Commands
--------------------------
- Dev: `npm run dev`
- Build: `npm run build`
- Start: `npm run start`
- Lint: `npm run lint`

Standard recovery when dev behaves weird:
1. Stop dev server
2. Delete `.next`
3. Optional: delete `node_modules/.cache`
4. Restart with `npm run dev`
